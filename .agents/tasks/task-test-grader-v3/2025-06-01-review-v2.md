**Watch for:**
- The file-size guard uses `if document.file_size and ...` — if Telegram ever sends `file_size=0` for a valid file, the guard is bypassed because `0` is falsy; practically unlikely for images but the conditional conflates "unknown size" with "zero size" (possible)
- Compressed-photo path (`receive_batch_photo`) still has no size guard at all; only the document path is protected. Telegram compresses photos to ~1-2MB so this is low-risk, but the asymmetry is worth noting (likely)
- The `/edit` recalculation silently skips sentinel answers (`"-"`, `"X"`) because they never match a correct letter — correct behavior, but a future maintainer adding partial-credit logic could break it without warning since there's no explicit sentinel check (possible)

## High-level view

The only remaining persistence gap is the intentional architectural deferral — unidentified results live only in bot memory until `/done` saves them.

The upload size guard rejects documents over 10MB before calling `get_file()`. The photo path lacks an equivalent guard; Telegram's server-side compression makes this acceptable on a 2GB instance.

Config logs `logger.warning(...)` for non-integer admin entries at import time, visible in the systemd journal on restart.

## Upload size guard: fail-open on unknown size

The `if document.file_size and ...` pattern means: if Telegram doesn't provide a file size (rare for documents but possible for forwarded files), the guard is bypassed and the file is downloaded regardless. This is fail-open for usability — rejecting files with unknown size would break legitimate uploads. The OpenCV decode that follows is bounded only by available memory, so the worst case is an OOM if someone sends a forwarded document without size metadata that happens to be enormous.

The guard sits before the download call but after a null-document check:

```python
if document.file_size and document.file_size > 10 * 1024 * 1024:
    ...
    return BATCH_SCANNING

file_name = (document.file_name or "").lower()
# ... extension check ...
file = await document.get_file()  # only reached if <10MB and valid extension
```

A 50MB `.pdf` is rejected by extension first; a 50MB `.jpg` is rejected by size before download.
