# Test grader bot V3: roster auto-naming, relative OMR, multi-admin

A major rewrite of the Telegram test grader bot. The OMR scanner replaces absolute fill thresholds with relative bubble comparison (darkest vs. second-darkest within a question), backed by a 5-stage fallback strategy for corner detection. The bot gains a roster system that auto-saves scanned sheets when the detected student number matches an entry, an `/edit` command that recalculates all scores on answer key changes, document-as-photo support in scan mode, and multi-admin access via comma-separated IDs. Session loading now gracefully handles corrupted JSON.

**Watch for:**
- Roster keys are stored as strings but looked up inconsistently between parse-time (int keys) and lookup-time (string conversion) — works now but fragile to future changes (confirmed)
- `/edit` recalculation compares answer letters against the `answers[]` array, which stores `"-"` for blanks and `"X"` for ambiguous — these can never equal a correct letter, so scoring is fine, but the pattern is implicit (confirmed)
- No file-size or image-dimension guard on document uploads — a user can send a 50MB PNG that OpenCV will try to decode in-memory on a constrained 2GB server (likely)
- Session file writes are not atomic — a crash mid-write corrupts the JSON and the graceful fallback resets the session, losing all student data (confirmed)

## High-level view

The OMR scanner is now threshold-agnostic at the bubble level: one adaptive threshold pass on the warped image, then per-question relative comparison decides which bubble is filled. The 5-stage corner detection cascades from standard Otsu through dilation, manual thresholds, 3-corner geometric estimation, and finally paper-boundary detection. Under heavy keystoning, the 3-corner parallelogram estimation can drift because it assumes a true parallelogram, not a perspective-distorted quadrilateral.

The bot's scanning flow splits at runtime: if a roster exists and the detected student number matches, results auto-save to disk immediately. Unmatched sheets queue in memory for manual naming at `/done`. This split means a bot restart mid-batch loses unidentified results while auto-saved ones survive — an asymmetric durability gap.

The `/edit` command modifies the answer key in-place and walks all stored student records to recalculate scores. The recalculation compares stored answer letters (including `"-"` and `"X"` sentinels) against the key — correct by accident of string inequality rather than by explicit sentinel handling.

Multi-admin is a list-membership check. The config parser silently drops non-integer values from the comma-separated list rather than failing at startup, which means a typo in the env var produces a silently reduced admin set.

## OMR relative detection and corner fallback chain

The scanner applies one `cv2.adaptiveThreshold` to the entire warped grayscale image, producing a single binary mask. Per-question detection compares fill ratios: the darkest bubble must exceed `RELATIVE_RATIO_THRESHOLD` (1.8x) times the second-darkest and clear `MIN_FILL_RATIO` (0.12). If neither condition holds, the question is flagged ambiguous and surfaced to the user as uncertain.

The 5-stage corner detection is ordered cheapest-first: Otsu, dilated Otsu, four manual thresholds (80/100/120/140), 3-corner parallelogram estimation, and paper-boundary contour. The 3-corner estimation uses `adj1 + adj2 - opposite` which is geometrically correct for a parallelogram but not for a perspective-distorted quadrilateral — under heavy keystoning, the estimated fourth corner drifts. The subsequent perspective transform may partially compensate, but scans taken at steep angles could produce misaligned grids. (likely)

## Session persistence and corruption handling

`_save_session` opens the file, truncates it, and writes JSON directly:

```python
with open(path, "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)
```

If the process is killed between open (which truncates) and the completed write, the file is left empty or partially written. On next load, the bot sees corruption and resets to `{}`, losing all student records. The fix is write-to-temp + `os.replace`:

```python
tmp = path.with_suffix('.tmp')
with open(tmp, "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)
os.replace(tmp, path)
```

Session files are written on every scan and every name entry — high frequency relative to the bot's lifetime.

## Batch scanning state split: memory vs. disk

Auto-saved results (student number matched roster) go directly into the session JSON on disk. Unidentified results live in `context.user_data["unidentified_results"]` — in-memory only. If the bot process restarts during a batch scan (e.g., systemd restart, OOM kill), the auto-saved results survive but all unidentified results vanish with no user-visible error. The next `/done` will see an empty unidentified list and report "all saved" even though sheets were lost.

A mitigation: persist unidentified results to a temporary file alongside the session, or always append to disk and mark records as "pending name" rather than splitting storage locations.

## Document upload without size constraints

`receive_batch_document` validates file extension but does not check `document.file_size`. The Telegram API provides this field before downloading. On a 2GB server running 3 bots, downloading and then `cv2.imread`-ing a large image (e.g., an uncompressed 20MP photo sent as document) could push memory into swap or trigger OOM. A guard like:

```python
if document.file_size and document.file_size > 10 * 1024 * 1024:
    await update.message.reply_text("Fayl hajmi juda katta (max 10MB).")
    return BATCH_SCANNING
```

would prevent this at minimal cost.

## grid_y layout coupling between scanner and generator

`scan_answer_sheet` computes the answer grid's Y offset with:

```python
grid_y = STUDENT_NUM_Y + 50 + 2 * STUDENT_NUM_SPACING_Y + 30 + 60
```

This formula must replicate the layout logic in `sheet_generator._draw_student_number_section`, which returns the computed `grid_start` value. If the generator changes row counts, spacing, or separator height, the scanner will read bubbles at wrong Y positions without any error signal. Exporting the grid start from the generator (or computing it from shared constants in a single place) would eliminate this silent coupling.

## Silent admin misconfiguration

The config parser swallows non-integer entries in `ADMIN_IDS` without any logging:

```python
try:
    ADMIN_IDS.append(int(part))
except (ValueError, TypeError):
    pass  # silently dropped
```

If an admin accidentally pastes a username or a truncated ID, the bot starts with fewer admins than intended and provides no diagnostic. Adding `logger.warning("Skipping invalid ADMIN_ID entry: %s", part)` in the except block would surface this at deploy time.
